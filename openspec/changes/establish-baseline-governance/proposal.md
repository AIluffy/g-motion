# Change: 建立基线治理与性能基线体系

## Why
当前重构计划涉及架构与性能层面的高风险变更，需要一个正式的变更治理与性能基线体系，以保证变更可追踪、可评审、可量化回归。

## What Changes
- 建立并正式启用 OpenSpec 变更流程，作为所有架构/性能/破坏性变更的强制前置门槛
- 盘点并固化核心 API 路径与关键性能指标的采集范围
- 以 Vitest bench + 现有基准脚本为基础，形成可复用的基线采集与报告流程
- 输出阶段 0 文档交付物：风险清单与性能基线报告

## Impact
- Affected specs: performance-baseline
- Affected code: 仅新增变更提案与基线文档，不修改运行时逻辑
